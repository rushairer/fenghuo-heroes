# 三国志列传：群英新篇开发计划

## 项目定位

原创三国题材 Web 策略游戏，技术栈为 Phaser + TypeScript + Vite。项目以《乱世群英》式经典 MD 三国策略体验为机制母版，尽可能复刻其机制、系统结构、操作流程、策略节奏和模块功能；素材、代码、音乐、角色头像、UI 美术、具体文案、战役编排、数值表、地图视觉、音效和剧情文本全部原创。

## 复刻边界

### 尽量复刻

- 游戏机制、系统结构、操作流程、策略节奏、模块功能。
- 君主命令、城池治理、内政、外交、军事、视察情况、月令推进、势力状态、行军出征、武将管理、战斗结算等体验骨架。
- 老派主机策略游戏的层级菜单、回合压力、资源取舍、出征前准备和战后反馈。

### 全面原创

- 素材、代码、音乐、角色头像、UI 美术、具体文案、战役编排、数值表、地图视觉、音效、剧情文本。
- 不逐像素复刻原作界面，不照搬图标、菜单样式、音乐旋律、角色造型、地图与文本。
- 目标是“功能同构 + 现代美术重制”，不是拷贝 ROM 或复刻原始资源。

## 当前可玩切片

- 标题、初始设定、君主选择、全国地图、城池命令、军事行动、月令报告已经形成主循环。
- 主界面已开始按月份分流：视察情况月显示内政、外交、军事三大主命令；行军月显示部队、移动、攻击、情报、撤退、待机、月令。
- 全国地图支持城池光标视察：点击任意城池可查看归属、区域、兵力、城防、府库、存粮；点击己方城池切换命令城，点击相邻敌城设为出征/外交目标。
- 内政命令绑定当前城池：开发、征兵、训练、治安、征粮会消耗政令并改变城池资源、民心、士气和情报。
- 外交命令绑定邻接势力：同盟、侦察、离间、劝降会影响目标势力行动、目标城守军、城防和情报。
- 人事系统城池化：武将有所在城，登用后进入当前城，本城任命、本城出战，并可向相邻己方城池移动。
- 军事流程开始收敛为“视察情况月下达出征令 → 行军月移动与攻击 → 进入会战分支 → 战后回写版图 → 月令报告”。
- 敌城情报页替代旧式战役说明：显示目标城归属、守军、城防、粮金、守将、邻接和胜算。
- 战斗守军按目标城和所属势力动态生成，不再固定为单一战役敌军。
- 战斗界面以军势为核心：显示我军/敌军总兵、将数、士气、补给、敌势和战场态势。
- 战斗命令使用编号窗口：移动、攻击、计略、待机、委任、撤退，并支持数字键操作。
- 战后结算会回写出发城兵力、目标城兵力、城防、缴获金粮和城池归属。
- 月令报告集中显示本月命令、己方收入、敌方行动、随机事件和新月份状态。
- 已加入曹操军、刘备军、孙权军、袁绍军、董卓军、群雄割据六类势力骨架，以及 15 座城池道路关系。
- 所有当前图片、头像、背景和程序化配乐均为原创占位资产。

## 原版复盘结论

详见 `ORIGINAL_GAME_STUDY.md` 与 `REPLICATION_BLUEPRINT.md`。当前复刻工作的最高优先级已经从“继续补现代战棋功能”调整为“恢复原版战略状态机”：

```text
视察情况月：城池视察 + 内政/外交/军事三栏命令
  -> 行军月：武将出城 + 路线移动 + 截粮/占村/攻城
  -> 攻城：强攻/围城/火计/挑战/撤退
  -> 单挑或会战
  -> 战果回写
  -> 月份交替
```

当前战棋只作为“会战”分支保留，不再作为最终战斗主干继续扩张。

## 复刻功能清单

1. 视察情况：城池、武将、兵力、粮草、金钱、民心、敌势。当前已有独立视察界面、战役总览、城池状态、武将任命、敌势和多势力版图。
2. 内政：开发、调动、情报、福利、任命、税率、教育、运输。当前仅有开发和若干近似命令，需要按原版矩阵归位。
3. 外交：同盟、计策、情报、借款、还款。当前已有同盟、侦察、离间、劝降，需要补暗杀、火计、债务和持续盟约。
4. 军事：征兵、武器、情报、人材、防卫、训练。当前军事菜单偏出征前准备，需要把征兵/训练/防卫/武器归入军事栏目。
5. 月令系统：每月资源变化、敌军行动、事件触发、本月命令记录与月令报告。
6. 战斗系统：武将带兵、军势总览、战场态势、地形、士气、兵种、编号战斗命令、委任、撤退、胜负结算。
7. 多势力版图：曹、刘、孙、袁、董与群雄城池归属、道路、势力目标。
8. 势力一览：各方实力对比、外交对象选择、战略威胁判断。
9. 人才搜索与登用：在野人物、举荐、失败线索、成功投效，并进入当前城池。
10. 人事与武将移动：本城任命、本城出战、相邻己方城池调动。
11. 游玩顺序界面：标题、剧本/年代/难度初始化、君主选择、全国地图选城、主命令、二级命令、战略执行、月令反馈。

## 标准游玩顺序

1. 标题：开始游戏、继续游戏、环境设定。
2. 初始设定：选择年代/剧本、难度、玩家数，确认后进入君主选择。
3. 君主选择：选择势力与君主；当前先开放刘备军，用于打磨完整流程。
4. 全国地图：查看城池归属、道路、敌我态势，点击己方城池作为当前命令对象。
5. 视察情况月：围绕当前城池执行内政、外交、军事三大栏目命令；这些命令只准备国力、外交、军备和情报。
6. 行军月：选择远征军，在全国地图上移动、截粮、占村、攻击城池或撤退。
7. 攻城/单挑/会战：到达敌城后先进入攻城层，必要时触发单挑或当前会战分支。
8. 月令循环：战果回写后，结算己方产出、敌方行动、随机事件，以月令报告进入下一个月份。

## 三大模块方向

### 内政

- 补给、征粮、整军、训练、伤兵恢复。
- 当前版本：征粮与整军会影响出征前资源和战斗属性。

### 外交

- 遣使、结盟、侦察、威慑、劝降。
- 当前版本：遣使提升士气，侦察削弱敌将初始状态。
- 当前版本：劝降会在情报足够时削弱目标城守军与城防。

### 军事

- 点将、编队、军令、阵型、目标情报、出征令、军事行动、战后处理。
- 当前版本：出征读取当前城池武将和相邻敌城，形成待执行出征令。
- 当前版本：军事行动按目标城动态生成守军，战后回写兵力、城防、缴获和归属。
- 当前版本：军令可在稳扎、奇袭、火攻之间切换，阵型可在方圆阵、先锋阵、雁行阵之间切换。

## 近期里程碑

1. 状态机纠偏：把当前 `inspection/march` 标签升级为硬流程，单数月进入视察情况，双数月进入行军。当前已完成第一版入口分流。
2. 菜单矩阵归位：内政、外交、军事按原版子命令重排，现有功能先复用，缺失项先给可执行占位和动画反馈。
3. 行军月 MVP：用远征军数据替代单薄的出征令，支持路线、移动、粮草、士气、目标城。
4. 攻城层 MVP：强攻、围城、火计、挑战、撤退，战棋降级为会战分支。
5. 单挑 MVP：攻击、防守、蓄势三按钮，结果回写攻城和武将状态。
6. 世界模拟：AI 远征军、同盟期限、债务、俘虏、继位、灭亡和统一判定。
7. 继续完成原创素材包：UI 纹理、单位小头像、战斗立绘、行军和单挑背景。
