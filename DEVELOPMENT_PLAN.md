# 三国志列传：群英新篇开发计划

## 项目定位

原创三国题材 Web 策略 RPG，技术栈为 Phaser + TypeScript + Vite。项目以《乱世群英》式经典 MD 三国策略体验为机制母版，尽可能复刻其机制、系统结构、操作流程、策略节奏和模块功能；素材、代码、音乐、角色头像、UI 美术、具体文案、关卡编排、数值表、地图视觉、音效和剧情文本全部原创。

## 复刻边界

### 尽量复刻

- 游戏机制、系统结构、操作流程、策略节奏、模块功能。
- 君主命令、城池治理、内政、外交、军事、视察情况、月令推进、势力状态、行军出征、武将管理、战斗结算等体验骨架。
- 老派主机策略游戏的层级菜单、回合压力、资源取舍、出征前准备和战后反馈。

### 全面原创

- 素材、代码、音乐、角色头像、UI 美术、具体文案、关卡编排、数值表、地图视觉、音效、剧情文本。
- 不逐像素复刻原作界面，不照搬图标、菜单样式、音乐旋律、角色造型、地图与文本。
- 目标是“功能同构 + 现代美术重制”，不是拷贝 ROM 或复刻原始资源。

- 月令推进已改为全版图城池产出，并提示整备最强的敌对势力兵力变化。
- 战斗胜利已接入版图结果：下邳攻略胜利后，下邳归入刘备军并影响地图颜色与势力统计。
- 敌方月令 AI 已接入：曹、孙、袁、董会屯兵备战或攻取邻接城池。
- 月令随机事件已接入：丰收、盗贼、名士传闻、边境告急会影响资源、民心、情报和敌势。

- 内政命令已按原版式菜单收敛为开发、征兵、训练、治安、征粮，并与当前城池、本月政令、城池资源绑定。
- 军事出征已从固定关卡改为读取当前城池的邻接敌城：选择目标后确认出征，战斗胜利会占领所选目标城。
- 外交交涉已加入对象选择：从当前城池周边势力中选择君主/势力，再执行同盟、侦察、离间、劝降。
- 外交结果已和地图状态联动：同盟会让目标势力暂缓攻刘，侦察会锁定邻接目标，离间/劝降会削弱目标城守军。
- 月令推进已升级为独立月令报告：集中显示收入、敌方行动、随机事件、新月份状态。
- 人事/武将已开始城池化：战役内武将表可变，登用后加入当前城池，本城任命只显示本城武将。
- 武将移动已加入人事命令：可消耗政令将本城武将调往相邻己方城池，出征也只读取当前城池武将。
- 本月命令簿已加入：内政、外交、人事调动、出征会记录到月令报告中，形成“下令 → 结算 → 下月”的回合节奏。

## 复刻功能清单

1. 视察情况：城池、武将、兵力、粮草、金钱、民心、敌势。当前已有独立视察界面、战役总览、城池状态、武将任命、敌势和多势力版图。
2. 内政：开发、征兵、训练、赏赐、治安、征粮。
3. 外交：同盟、劝降、侦察、离间、交涉。当前已有独立外交交涉界面、邻接势力对象选择、成功率结算和地图状态影响。
4. 军事：出征、移动、攻击、编队、任命、撤退。当前已有任命、邻接目标选择、出征确认、粮草消耗和战棋移动攻击。
5. 月令系统：每月资源变化、敌军行动、事件触发、本月命令记录与月令报告。
6. 战斗系统：武将带兵、地形、士气、兵种、胜负结算。
7. 多势力版图：曹、刘、孙、袁、董与群雄城池归属、道路、势力目标。
8. 势力一览：各方实力对比、外交对象选择、战略威胁判断。
9. 人才搜索与登用：在野人物、举荐、失败线索、成功投效，并进入当前城池。
10. 人事与武将移动：本城任命、本城出战、相邻己方城池调动。
11. 游玩顺序界面：标题、剧本/年代/难度初始化、君主选择、全国地图选城、主命令、二级命令、战略执行、月令反馈。

## 标准游玩顺序

1. 标题：开始游戏、继续游戏、环境设定、开发计划。
2. 初始设定：选择年代/剧本、难度、玩家数，确认后进入君主选择。
3. 君主选择：选择势力与君主；当前先开放刘备军，用于打磨完整流程。
4. 全国地图：查看城池归属、道路、敌我态势，点击己方城池作为当前命令对象。
5. 城池命令：围绕当前城池执行视察、内政、外交、军事、人事、机能。
6. 军事行动：从当前城池选择出征、编队、任命、情报等操作，进入战斗或战前准备。
7. 月令循环：结算己方产出、敌方行动、随机事件，以月令报告进入下一个月份。

## 三大模块方向

### 内政

- 补给、征粮、整军、训练、伤兵恢复。
- 当前版本：征粮与整军会影响出征前资源和战斗属性。

### 外交

- 遣使、结盟、侦察、威慑、劝降。
- 当前版本：遣使提升士气，侦察削弱敌将初始状态。
- 当前版本：劝降会在情报足够时削弱普通伏兵。

### 军事

- 点将、编队、出征、战术、战后封赏。
- 当前版本：点将查看阵容，出征进入小沛东道战斗。
- 当前版本：军令可在稳扎、奇袭、火攻之间切换，阵型可在方圆阵、先锋阵、雁行阵之间切换。

## 美术与音频策略

- 所有头像和背景使用原创生成图，不使用现有游戏截图或临摹式复刻素材。
