# 《乱世群英》菜单功能扫尾计划

本文件用于把复刻工作从“想到哪里做哪里”收敛为“按原版菜单逐项验收”。执行顺序以本文件为准；每次改动只推进少量菜单项，并同步更新状态。

## 总原则

- 每个功能必须先归属到原版栏目：内政、外交、军事、行军、攻城、单挑、月令。
- 每个命令必须有固定流程：选择命令 -> 选择或确认发起方 -> 选择或确认目标 -> 显示效果 -> 确认执行 -> 播放反馈 -> 回到当前月份界面。
- 不能再新增“按钮直接生效”的旧式入口。
- 城池、武将、部队、势力四类对象要分清。能指定武将的命令不要只改城市总数值。
- 当前阶段允许数值和动画简化，但输入、目标、结果方向必须先对齐。

## 外交扫尾

| 命令 | 当前状态 | 扫尾动作 | 优先级 |
|---|---|---|---|
| 同盟 | 有一次性效果和确认 | 增加同盟期限、通道、破盟惩罚 | P1 |
| 计策·离间 | 有削兵/降敌势 | 目标改为城/武将；接入忠诚与失败暴露 | P1 |
| 计策·暗杀 | 有扰乱守备 | 接入目标守将、伤病/被俘结果 | P2 |
| 计策·火计 | 有烧粮/降城防 | 接天气/情报/军师影响；可用于攻城前置 | P1 |
| 情报 | 有确认和情报加成 | 统一显示目标势力/城池详情 | P1 |
| 借款 | 有一次性加金 | 增加债务对象、金额、到期月 | P2 |
| 还款 | 有一次性效果 | 清除债务并影响外交信用 | P2 |

验收：外交命令的目标必须是邻接势力或敌城，不允许模糊地改全局敌势。

## 执行顺序

1. 军事扫尾：征兵规模、训练目标、出征选择随军武将和兵粮。当前已完成征兵三档、训练单将/全军目标、随军武将和随军粮档位。
2. 行军扫尾：路线、多段移动、部队详情、攻城距离判断。
3. 攻城扫尾：会战战损回写武将、围城回合、挑战守将。
4. 内政扫尾：税率档位、运输到相邻城、调兵调粮。
5. 外交扫尾：同盟期限、债务、计策目标武将。
6. 月令扫尾：所有持续状态进入月令结算。

## 每次提交验收问题

- 这个功能属于哪个原版菜单？
- 发起方是谁？
- 目标是谁？
- 输入是什么？
- 结果回写到城池、武将、部队还是势力？
- 是否仍存在绕过确认流程的旧入口？
- 文档状态是否同步更新？
