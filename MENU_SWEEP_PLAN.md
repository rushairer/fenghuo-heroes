# 《乱世群英》菜单功能扫尾计划

本文件用于把复刻工作从“想到哪里做哪里”收敛为“按原版菜单逐项验收”。执行顺序以本文件为准；每次改动只推进少量菜单项，并同步更新状态。

命令对象选择的详细底表见 `COMMAND_FLOW_AUDIT.md`。本文件负责扫尾排期；`COMMAND_FLOW_AUDIT.md` 负责逐条定义每个命令的发起方、目的方、目标数量和当前项目差距。

## 总原则

- 每个功能必须先归属到原版栏目：内政、外交、军事、行军、攻城、单挑、月令。
- 每个命令必须有固定流程：选择命令 -> 选择或确认发起方 -> 选择或确认目标 -> 显示效果 -> 确认执行 -> 播放反馈 -> 回到当前月份界面。
- 不能再新增“按钮直接生效”的旧式入口。
- 城池、武将、部队、势力四类对象要分清。能指定武将的命令不要只改城市总数值。
- 当前阶段允许数值和动画简化，但输入、目标、结果方向必须先对齐。
- 地图点击只允许更新光标、默认命令城和视察对象；真正执行命令时，内政、外交、军事都必须进入统一对象选择/确认流程，不能一个栏目从地图取目标、另一个栏目从菜单取目标。
- 菜单功能收敛后进入 UI 系统梳理：多级菜单统一使用半透明遮罩、投影、顶部层次条和自适应高度；避免面板压住底部主操作区或遮挡关键信息。

## 外交扫尾

外交不再保留常驻“外交对象”选择框。所有外交命令统一走：选择外交子命令 -> 选择出使城/发起方 -> 选择邻接势力或敌城目标 -> 显示发起方、目标、范围、效果 -> 确认执行。

| 命令 | 当前状态 | 扫尾动作 | 优先级 |
|---|---|---|---|
| 同盟 | 已按统一流程选择出使城和目标势力，成功后生成4月盟约并在月令递减 | 后续补通道、破盟惩罚和同盟期内命令限制 | P1 |
| 计策·离间 | 已按统一流程选择出使城、邻接敌城、城内武将；成功回写武将忠诚/兵力、城池兵力和敌势 | 后续接军师影响、低忠倒戈和失败暴露动画 | P1 |
| 计策·暗杀 | 已按统一流程选择出使城、邻接敌城、城内武将；成功扰乱目标部曲并回写城防/兵力 | 后续接伤病/被俘状态和守将优先规则 | P2 |
| 计策·火计 | 已按统一流程选择发起城和目标敌城，有烧粮/降城防 | 接天气/情报/军师影响；可用于攻城前置 | P1 |
| 情报 | 已按统一流程选择出使城和目标敌城，成功后显示势力、城池、武将和邦交报告 | 后续让情报不足时显示区间，接军师影响 | P1 |
| 借款 | 已按统一流程选择出使城和目标势力，生成6月债契，到期未清进入月令惩罚 | 后续增加金额选择、信用和债务谈判 | P2 |
| 还款 | 已按统一流程选择出使城和邻接债权势力，还款递减或清除债契 | 后续增加金额选择和逾期罚息 | P2 |

验收：外交命令的目标必须是邻接势力或敌城，不允许模糊地改全局敌势。

## 执行顺序

1. 军事扫尾：征兵规模、训练目标、出征选择随军武将和兵粮。当前已完成征兵三档、训练单将/全军目标、随军武将、随军粮档位和逐将兵粮明细。
2. 行军扫尾：路线计划驱动和逐段移动已完成首轮；继续手动途经节点、部队详情和节点事件。
3. 攻城扫尾：会战战损回写武将已完成首轮；继续围城回合、挑战守将、俘虏/伤病。
4. 内政扫尾：税率档位、运输金/兵、调兵调粮。当前运输已支持远征粮仓和相邻己方城。
5. 外交扫尾：计策目标武将、同盟期限、债务首轮已完成；继续军师/情报影响、破盟和金额选择。
6. 月令扫尾：税率、债务、同盟已进入月令；继续太守/军师、各城民心和外交信用。
7. UI 系统梳理：主命令菜单和确认面板已开始统一；继续替换对象选择层与长内容页面。
