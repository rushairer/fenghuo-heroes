# 《乱世群英》命令流程复盘与项目对照表

本文件是菜单扫尾的执行底表。每次修改命令流程前，先查本文件；每次改完，必须同步更新“当前项目状态”和“下一步修改”。

## 资料边界

已查证资料能确认原版的宏观结构：标题/初始化/君主选择后进入视察情况与行军循环；视察情况包含内政、外交、军事三类命令；行军阶段进行武将出城、路线移动、截粮、占村、攻城和单挑等行为。

公开资料能确认的子命令清单：

- 内政：开发、调动、情报、福利、任命、税率、教育、运输
- 外交：同盟、计策、情报、借款、还款，其中计策含离间、暗杀、火计
- 军事：征兵、武器、情报、人材、防卫、训练

本文件中更细的“对象选择顺序”和“约束类型”是基于上述清单、原版体验复盘、现有项目规则的归纳设计。后续若拿到更精确说明书或实机录像，应以证据修正本表。

## 统一命令协议

项目后续不允许再出现“内政从地图选目标、外交从菜单选目标、军事从另一套面板选目标”的割裂形态。地图只负责光标、视察、当前命令城提示；真正执行命令时，必须进入统一命令协议。

标准流程：

1. 选择主栏目：内政、外交、军事，或进入行军、攻城、单挑层。
2. 选择子命令。
3. 进入统一对象选择器，根据该命令的对象规格依次选择或确认对象。
4. 显示确认面板：发起方、目的方/目标、范围、消耗、效果、风险。
5. 确认执行。
6. 播放反馈/动画占位。
7. 回到当前层：视察情况、行军、攻城或月令报告。

对象选择器必须支持这些形态：

| 形态 | 含义 | 示例 |
|---|---|---|
| A0 | 无额外目标，只确认当前城/当前军 | 福利、防卫、行军待机 |
| A1 城 | 单一城市目标 | 开发当前城、税率当前城、情报敌城 |
| A1 武将 | 单一武将目标 | 征兵给某武将、配武器、任命职位 |
| A1 势力 | 单一势力目标 | 同盟、借款、还款 |
| A1 部队 | 单一远征军目标 | 行军移动、撤退 |
| A2 城->城 | 发起城和目的城 | 调动、运输、出征目标 |
| A2 城->势力 | 发起城和目的势力 | 同盟、借款、还款 |
| A2 城->敌城 | 发起城和目标敌城 | 外交情报、火计、劝降、军事情报 |
| A2 城->武将 | 发起城和本城武将 | 征兵、武器、训练、教育 |
| A3 城->城->对象 | 两城之间再选移动对象/数量 | 调动武将、调兵、调粮、运输金粮兵 |
| A3 城->敌城->武将 | 敌城里再选守将/目标武将 | 离间、暗杀、挑战守将 |
| Multi | 多选武将或资源项 | 出征随军武将、调动多名武将 |

统一输入原则：

- 当前命令城可以作为默认发起方，但确认面板前必须能看见或切换发起方。
- 任何命令都不能依赖“刚才地图上点过哪个敌城”作为隐式目标。
- 地图点击可以更新光标和推荐目标，但命令执行必须经过对象选择器。
- 能落到武将的命令必须落到武将，不能只改城市总数。
- 能落到城池的命令必须明确城池，不能只改全局敌势。

## 当前最高优先级差距

1. 对象选择器不统一：内政、外交、军事虽然都有确认，但入口和对象来源不同。
2. 地图光标和命令目标混用：点地图有时等于切换目标，有时只是视察。
3. 内政多数命令仍默认当前城，缺少显式发起方选择或确认。
4. 外交已改出使城/目标流程，离间/暗杀已细化到敌城内武将，同盟期限和债务已进入月令；剩余为破盟、信用、金额选择。
5. 军事征兵/武器/训练已能选武将，但发起城选择仍隐含。
6. 调动、运输、出征都需要 A3/Multi 选择器，否则数量和对象会继续混乱。

## 落盘推进办法

每个小闭环必须按这个顺序执行：

1. 在本文件中选一条命令。
2. 更新该命令的“当前项目状态/下一步修改”。
3. 修改代码。
4. 运行 `npm run build`。
5. 更新 `MENU_SWEEP_PLAN.md` 的对应状态。
6. 提交并推送。

推荐执行顺序：

1. 建立统一对象选择器骨架，先不改变数值逻辑。
2. 把内政“开发、福利、税率、防卫”这类 A1 城命令纳入统一确认/发起城选择。
3. 把军事“征兵、武器、训练”纳入 A2 城->武将流程。
4. 把外交计策补到 A3 城->敌城->武将。
5. 把调动、运输、出征补到 A3/Multi。
6. 税率、债务、同盟期限已进入月令；继续补太守/军师影响、金额选择和各城独立民心。
