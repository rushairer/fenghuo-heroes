# 《三国志列传：乱世群英》原版复盘档案

本文件是后续复刻工作的“事实底座”。任何新增功能都应先映射到这里的原版结构，再进入实现。

## 目标

- 复刻对象：世嘉 Mega Drive《三国志列传：乱世群英》的机制、流程、菜单结构、策略节奏和关键体验。
- 原创边界：代码、图片、头像、音乐、音效、UI 纹理、具体文本、数值表和剧情表现全部原创。
- 方法：学习原版的系统架构和操作关系，而不是复制原版素材、画面或文本。

## 已查证事实

来源包括公开条目、玩家研究帖、视频记录和说明书资料入口。后续需要用实机/视频逐帧验证。

1. 基础定位

- 1991 年，SEGA 在 Mega Drive 平台发行。
- 类型为回合制策略游戏。
- 背景为三国时代，目标是统一中国全境。
- 玩家最多可同时控制 3 个势力。

参考：
- https://zh.wikipedia.org/wiki/三國志列傳_亂世群英
- https://www.233leyuan.com/game-detail/2444618

2. 原版不是普通“每月同一张菜单”的结构

- 原版明确区分「视察情况」与「行军」两种面。
- 「视察情况」在单数月执行。
- 「行军」在双数月执行。
- 这意味着我们的“每月循环”不能只做成现代 SLG 的统一月令按钮，而应逐步恢复为视察月/行军月交替。

参考：
- https://zh.wikipedia.org/wiki/三國志列傳_亂世群英

3. 视察情况面的三大栏目

原版核心栏目是：

- 内政
- 外交
- 军事

公开资料列出的子指令如下，需继续核验中文 MD 版具体显示名和排列：

内政：
- 开发
- 调动
- 情报
- 福利
- 任命
- 税率
- 教育
- 运输

外交：
- 同盟
- 计策：离间、暗杀、火计
- 情报
- 借款
- 还款

军事：
- 征兵
- 武器
- 情报
- 人材
- 防卫
- 训练

参考：
- https://zh.wikipedia.org/wiki/三國志列傳_亂世群英

4. 行军面是原版灵魂之一

- 玩家选择武将后从城池派出。
- 设定路线后，可在中国地图上移动。
- 可攻击他国城池。
- 公开资料提到自由行军、截粮、占村。
- 因此本项目的“攻击”应先进入攻城决策层，再视情况触发单挑或会战，不能由会战直接替代攻城。

参考：
- https://zh.wikipedia.org/wiki/三國志列傳_亂世群英
- https://www.233leyuan.com/game-detail/2444618

5. 单挑系统是原版辨识度很高的特色

- 原版有可操控武将一对一战斗的单挑系统。
- 这不能被普通战棋或自动结算完全替代。
- 我们当前的格子战斗只是临时切片，后续应向“行军遭遇/攻城/单挑/军势结算”的组合靠拢。

参考：
- https://zh.wikipedia.org/wiki/三國志列傳_亂世群英

6. 玩家研究中提到的重要机制线索

待验证但优先级高：

- 行军路线会影响武将成长。
- 电脑会对玩家使用计策。
- 计策可能成功，也可能失败并导致武将被俘。
- 君主死亡后存在继位规则。
- 一个势力可能有多次继位机会，继位耗尽后势力灭亡。

参考：
- https://tieba.baidu.com/p/8285017559

## 当前项目与原版差距

1. 最大结构差距

- 当前项目仍是“全国地图 + 每月统一菜单”的现代化结构。
- 原版应恢复为「视察情况月」与「行军月」交替。

2. 菜单差距

- 当前已有三大主栏目，但子指令还没有完全对齐原版。
- 内政、外交、军事子项需要按原版清单重新整理。

3. 行军差距

- 当前“出征令 -> 战斗”过于直接。
- 原版行军面需要路线设定、地图移动、截粮、占村、攻城等过程。

4. 战斗差距

- 当前战斗仍是格子战斗切片。
- 原版重点应包含行军遭遇、攻城结算和可操作单挑。
