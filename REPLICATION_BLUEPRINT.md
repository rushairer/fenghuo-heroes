# 《乱世群英》机制复刻蓝图

本文件把原版研究转成可执行工程路线。后续实现优先级以这里为准。

## 一句话方向

当前项目不能继续按“现代全国地图 + 每月统一命令 + 直接战棋”推进。主干应改为：

```text
开局设定
  -> 视察情况月：城池视察 + 内政/外交/军事三栏命令
  -> 行军月：武将出城 + 路线移动 + 截粮/占村/攻城
  -> 攻城：强攻/围城/火计/挑战/撤退
  -> 单挑或会战
  -> 战果回写
  -> 月份交替
```

## 原版硬骨架

已查证内容：

- 原版是 1991 年 SEGA Mega Drive 回合制策略游戏，目标是统一中国全境。
- 原版分为「视察情况面」和「行军面」。
- 视察情况面可执行内政、外交、军事三类指令。
- 行军面可选择武将从城池派出、设定路线、在中国地图移动或攻击他国城池。
- 原版有可操作武将单挑系统。

主要参考：

- https://zh.wikipedia.org/wiki/三國志列傳_亂世群英
- https://www.233leyuan.com/game-detail/2444618

## 最高优先级差距

| 模块 | 当前状态 | 原版方向 | 优先级 |
|---|---|---|---|
| 月份状态机 | `inspection/march` 只是标签，菜单仍每月通用 | 单数月视察情况，双数月行军，命令入口完全不同 | P0 |
| 行军 | 当前是选择邻接目标后直接进入战斗 | 武将出城、设路线、在地图移动、截粮、占村、攻城 | P0 |
| 攻城 | 当前由战棋胜负直接占城 | 先有强攻/围城/火计/挑战/撤退等攻城层 | P0 |
| 单挑 | 缺失 | 独立可操作武将一对一，结果回写士气、伤亡、俘虏 | P0 |
| 菜单矩阵 | 子项是现代混合菜单 | 内政/外交/军事子项按原版结构重排 | P1 |
| AI | 敌方直接涨兵或吞并邻城 | AI 也应有内政、外交、计策、行军军团 | P1 |
| 世界模拟 | 有势力和城池骨架 | 灭亡、继位、统一判定、债务/同盟/俘虏 | P1 |

## 状态机设计

建议逐步把当前 `Phase` 改成更清晰的 `GamePhase`：

```ts
type GamePhase =
  | 'title'
  | 'scenarioSetup'
  | 'rulerSelect'
  | 'inspectionMonth'
  | 'inspectionCommand'
  | 'marchMonth'
  | 'marchMap'
  | 'siegeSetup'
  | 'siegeAction'
  | 'duel'
  | 'fieldBattle'
  | 'monthReport'
  | 'result'
```

日历应驱动 UI，而不是只显示标签：

```ts
type CampaignCalendar = {
  year: number
  month: number
  monthKind: 'inspection' | 'march'
  activeFactionId: FactionId
  actionPoints: number
}
```

## 行军数据模型

`PendingExpedition` 应升级为真正的远征军：

```ts
type MarchArmy = {
  id: string
  factionId: FactionId
  sourceCityId: CityId
  targetCityId?: CityId
  leaderOfficerId: string
  officerIds: string[]
  troops: number
  food: number
  morale: number
  position: {
    kind: 'city' | 'route' | 'field'
    cityId?: CityId
    route?: [CityId, CityId]
    progress?: number
  }
  routePlan: CityId[]
  movePoints: number
  status: 'ready' | 'marching' | 'besieging' | 'retreating' | 'routed'
}
```

行军月可用命令：

- 移动：沿路线推进，消耗移动点和粮。
- 查看：显示部队武将、兵力、粮草、士气、路线。
- 截粮：邻近敌方粮道或村落时获得粮，可能降名声或触发敌军。
- 占村：获得补给或情报，可能影响民心。
- 攻击：对邻接敌城进入攻城层。
- 撤退：沿来路返回，损士气。
- 待机：恢复少量士气或等待下月。

## 攻城与单挑

攻城状态：

```ts
type SiegeState = {
  id: string
  attackerArmyId: string
  defenderCityId: CityId
  phase: 'approach' | 'choice' | 'resolve' | 'duelOffer' | 'fieldBattle' | 'completed'
  wallHp: number
  defenderTroops: number
  attackerTroops: number
  turns: number
  lastAction?: 'assault' | 'surround' | 'fire' | 'challenge' | 'retreat'
}
```

攻城指令：

- 强攻：高伤害、高兵损、降城防。
- 围城：低兵损，消耗双方粮，削士气。
- 火攻：消耗情报/计略，降粮草或城防，受天气影响。
- 挑战：进入单挑，风险高但可能快速破局。
- 撤退：保留部队，损士气和粮。
- 会战：必要时进入当前 Phaser 战场，作为攻城分支而不是唯一入口。

单挑状态：

```ts
type DuelState = {
  attackerOfficerId: string
  defenderOfficerId: string
  attackerHp: number
  defenderHp: number
  attackerStamina: number
  defenderStamina: number
  round: number
  log: string[]
  outcome?: 'attackerWin' | 'defenderWin' | 'draw' | 'capture' | 'death'
}
```

单挑 MVP 命令：

- 攻击：高伤害，高体力消耗。
- 防守：低伤害，减伤并回体。
- 蓄势：下回合增强，可能被克制。

单挑结果回写：

- 胜方部队士气上升。
- 败方武将受伤、被俘、撤退或阵亡。
- 守将败北可能直接导致城防崩溃或守军溃散。

## 工程实施路线

### 阶段 1：状态机纠偏

- 新增 `inspectionMonth` 和 `marchMonth` 入口。
- `advanceCampaignMonth()` 按奇偶月进入不同 UI。
- 视察月显示内政/外交/军事。
- 行军月显示部队/移动/攻击/撤退/结束行军。
- 暂时保留现有战斗，降低风险。

### 阶段 2：菜单矩阵归位

- 内政显示开发、调动、情报、福利、任命、税率、教育、运输。
- 外交显示同盟、计策、情报、借款、还款。
- 军事显示征兵、武器、情报、人材、防卫、训练。
- 现有功能先复用，缺失项先给可执行占位和结果动画。

### 阶段 3：行军月 MVP

- 用 `MarchArmy` 替换 `PendingExpedition`。
- 视察月只能编成远征军。
- 行军月在地图上移动远征军。
- 到达敌城先进入攻城入口。

### 阶段 4：攻城层

- 新增 `SiegeState`。
- 实现强攻、围城、火攻、撤退。
- 战果先数值结算，当前战棋作为“会战”分支。

### 阶段 5：单挑 MVP

- 新增 `DuelState` 和单挑界面。
- 三按钮操作：攻击、防守、蓄势。
- 结果回写攻城和武将状态。

### 阶段 6：世界模拟

- AI 生成远征军，而不是直接改城市归属。
- 增加同盟期限、债务、俘虏、继位、灭亡和统一判定。

## 近期不做

- 不逐像素复刻原版画面。
- 不照搬原版素材、音乐、文本、地图视觉。
- 不继续扩展现代战棋技能树。
- 不把当前战棋当成最终战斗主干。
